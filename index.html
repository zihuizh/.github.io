上面示例里面，factorial的docstring里面的就是测试代码。仔细看的话会不会觉得这些测试代码很眼熟。

没错，doctest的测试代码就跟Python交互式控制台（命令行里输入python跑起来的那个）里的打印一样。

所以其实Doctest的测试原理是把我们在Python控制台的输入输出记录保存到函数的docstring里，然后一一把这些输入到解析器然后对比输出是否一致用来确定测试结果是否通过。

说到这里其实docstring基本都讲完了，为什么?

因为你只需要把自己平时在Python交互式控制台上测代码时的输入输出记录，拷贝到docstring里，以后要回归这个函数的时候就可以通过doctest回放测试用例了。至于Python交互式控制台怎么用、怎么测代码，这个跟呼吸吃饭拉屎睡觉一样就不讲了。

1.3 Defining New Functions

Web

  • 编程语言中所具有的特性：
    1. 数字和四则运算符是基础的内置数据与函数。
    2. 函数之间的嵌套提供了组合运算符的方法。
    3. 将名称与数值绑定在一起提供了一种有限的抽象方法。
  • 定义函数——一种更强大的抽象方法，可以将名字（函数名）与一系列运算绑定在一起，并看作一个单元。
  • How to define a function :
    def <name>(<formal parameters>): return `
    使用<def>语句来定义函数，包括函数名<name>以及使用句号分割的形参表<formal parameters>；在函数体的最后，使用<return>语句返回数值（要计算的表达式）。
    四个空格的缩进是必须的，用以表示层级关系。
  • 定义函数与赋值都可以为名字绑定新值，原有的关系会被破坏。[[00-CS61A#Video 2]]

1.3.1 Environments

  • 环境(Environment)是由一系列结构(frame)组成的。每一个结构中都包含了绑定关系，将名字同值联系起来。环境中只有一个全局结构。
  • 环境图(Environment diagrams)中展示了环境中各个结构以及结构中名字同值的关系。Online Python Tutor
    • 从中可以看出<import>以及<def>语句可以将名字同函数固有名称(intrinsic name)联系起来。
    • 环境图中左侧为名称，右侧为值（包括数字、数组、函数固有名称），名称可以变换与值的绑定关系，右侧的函数固有名称则不变。
    • 一个函数可以有多个函数名，但这个函数本身只有一个固有名称。
    • 函数签名：函数形参的描述。

1.3.2 Calling User-Defined Functions

在执行一个函数时，Python首先会计算运算符以及参数表达式，计算完成后再带入函数当中。[[00-CS61A#Control Evaluation Rule for Call Expression]]
执行函数时会形成一个**本地结构(local frame)**：

  • 在这一新的结构中的形参绑定输入的参数。
  • 在这一新的结构中执行函数体。
    函数应用的每一个实例（函数每被执行一次）都会形成对应的local frame。
    A name evaluates to the value bound to that name in the earliest frame of the current environment in which that name is found.
    这句话的含义是一个名字所代表的值是当前环境中最早找到的那一个。 例如在这样一段程序中，在global frame中，square是一个函数，在square的local frame中，square是2。那么在return mul(square,square)中，square代表2，因为在local frame中这是距离最近的值了。同样的，mul就代表函数，因为在local frame 中找不到对应的值，而在global frame中就找到是一个函数。

1.3.3 Example: Calling a User-Defined Function

首先执行sum_square(5,12)，生成local frame，其中x=5，y=12。
对于add(square(x), square(y))，执行add()首先要计算参数值，首先从括号最内部左侧的square(x)开始，在square新建的square local frame中再次将x绑定为5，计算并返回25，之后类似地可以得到144。
在add local frame中，可以计算得到189。

  • 名称与值绑定。
  • 函数每被执行一次就会生成一个本地结构（作用域）。

1.3.4 Local Names

函数的形参名称不影响函数的行为（因为最终都会与值绑定）。
这就要求形参名称是局部变量，这样不同函数之间不会相互影响。
同时，函数体内部的本地名称作用域也仅仅局限在函数体内部。

1.3.5 Choosing Names

命名指南：

  1. 函数名为小写，单词用下划线分隔。鼓励使用描述性名称。
  2. 函数名通常对应解释器应用于参数的操作（例如：print、add、square）或结果量的名称（例如：max、abs、sum）。
  3. 参数名称为小写，单词用下划线分隔。最好使用单字名称。
  4. 参数名称应该同时反映参数在函数中的角色，而不仅仅反映允许的参数类型。
  5. 单字母参数名称在其作用明显时可以接受，但应避免使用“l”(小写L)、“O”或“I”(大写i)，以避免与数字混淆。

1.3.6 Functions as Abstractions

函数的定义应该能够隐藏细节。函数的用户可能没有自己编写函数，是从另一个程序员那里获得的“黑盒”。程序员不需要知道如何实现函数，但仍能使用它。Python库具有此属性，许多开发人员使用这里定义的函数，但很少关心它们如何实现。
我们关心函数的以下属性：

  • 定义域
  • 值域
  • 目的（参数与输出值的关系）
    但并不关心函数内部如何实现。

List Comprehensions

[<map exp> for <name> in <iter exp> if <filter exp>]
Short version: [<map exp> for <name> in <iter exp>]
A combined expression that evaluates to a list using this evaluation procedure:

  1. Add a new frame with the current frame as its parent
  2. Create an empty result list that is the value of the expression
  3. For each element in the iterable value of <iter exp>:
    A. Bind <name> to that element in the new frame from step 1
    B. If <filter exp> evaluates to a true value, then add the value of <map exp> to the result list

Trees

Our tree abstract data type consists of a root and a list of its branches. To create a tree and access its root value and branches, use the following constructor and selectors:

  • Constructor
    • tree(label, branches=[]): creates a tree object with the given label value at its root node and list of branches. Notice that the second argument to this constructor, branches, is optional - if you want to make a tree with no branches, leave this argument empty.
  • Selectors
    • label(tree): returns the value in the root node of tree.
    • branches(tree): returns the list of branches of the given tree.
  • Convenience function
    • is_leaf(tree): returns True if tree‘s list of branches is empty, and False otherwise.

Recursive Functions

  • Definition: A function is called recursive if the body of that function calls itself,
    either directly or indirectly.
    就类似[[2-1 Environment Diagrams#Self-Reference]]一样 The Anatomy of a Recursive Function
  • The def statement header is similar to other functions
  • Conditional statements check for base cases
  • Base cases are evaluated without recursive calls
  • Recursive cases are evaluated with recursive calls

Recursive in Environment Diagrams

  • The same function fact is called multiple times
  • Different frames keep track of the different arguments in each call
  • What n evaluates to depends upon the current environment
  • Each call to fact solves a simpler problem than the last: smaller n
    Iteration(迭代) is a special case of recursion(递归)
    迭代：
    $$n!=\prod_{k=1}^nk$$
    递归：
    $$n!=\left{\begin{align}&1\quad &if\ \ n=0\
    &n*(n-1)!\quad &otherwise
    \end{align}\right.$$
    两种表达方式。

Verifying Recursive Functions

Is fact implemented correctly?

  1. Verify the base case
  2. Treat fact as a functional abstraction!
  3. Assume that fact(n-1) is correct
  4. Verify that fact(n) is correct
    类似数学归纳法

Order of Recursive Calls

  • Each cascade frame is from a different call to cascade.
  • Until the Return value appears, that call has not completed.
  • Any statement can appear before or after the recursive call.
    通过打印的顺序可以发现，递归的执行顺序是从外面剥洋葱，剥到核心后，在从里面把洋葱皮一层层盖上，剥洋葱就是函数里面迭代语句之前的部分，盖上就是后面的部分，所以才有这种输出。
